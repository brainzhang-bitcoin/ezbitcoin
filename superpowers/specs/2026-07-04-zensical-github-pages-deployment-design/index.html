Spec: EzBitcoin Docs Zensical Static Site & GitHub Pages Deployment¶

This spec defines the organization of the EzBitcoin article library using Zensical and the automated build and deployment process to GitHub Pages with a custom domain.

Metadata¶

  • Status: APPROVED
  • Date: 2026-07-04
  • Target URL: https://ezbitcoin.brainz.fun/
  • SSG Engine: Zensical
  • Deployment Platform: GitHub Pages (automated via GitHub Actions)

1. Zensical Site Configuration (zensical.toml)¶

The zensical.toml file will be created in the repository root to govern the site build, theme colors, navigation layout, and global options.

General Options¶

  • site_name = "EzBitcoin - 极简比特币讲解"
  • site_url = "https://ezbitcoin.brainz.fun/"
  • docs_dir = "docs"

Theme Options¶

  • Using the classic material theme.
  • Primary color: indigo
  • Accent color: indigo
  • Light/Dark auto-adaptive schemes enabled.
  • Key navigation and search features:
  • navigation.tabs
  • navigation.sections
  • navigation.top
  • search.suggest
  • search.highlight

Navigation Menu ([nav])¶

The documentation structure will be flattened into 5 primary top-level tabs, putting the Lightning Network section at the end:

  1. 主页
  2. 新手指南 (Beginners Guide and initial guides)
  3. 技术核心 (Technical core, blockchain, cryptography, keys, transactions, script, CS concepts)
  4. 网络与挖矿 (Mining, network, and upgrade protocols)
  5. 闪电网络 (Lightning network and LND-related technical guides)

3. GitHub Actions Workflow Configuration¶

To automate the compilation and deployment, a GitHub Actions workflow will be added.

Workflow File Path¶

.github/workflows/deploy.yml

Permissions Required¶

  • permissions.contents: write to allow the Action runner to commit and push build assets to the gh-pages branch.

Step-by-Step Pipeline¶

  1. Check out repository code.
  2. Set up Python (version 3.12).
  3. Install base dependencies from requirements.txt and install zensical via pip.
  4. Execute zensical build to compile Markdown documentation to static HTML in the ./site directory.
  5. Deploy ./site to GitHub Pages using peaceiris/actions-gh-pages with the custom domain parameter cname: ezbitcoin.brainz.fun.

4. Verification & Testing¶

  • Local Preview: Validate by running zensical serve locally and verifying that the nav tabs, page layouts, and search functionality render correctly.
  • Workflow Simulation: Verify that the yaml syntax is valid.
